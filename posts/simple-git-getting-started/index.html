Simple git: Getting started

This is a collection of Git commands and short explanations of what they do. It should have you using the most common 80% of Git functionality chop chop.

Phrases that are marked like this are commands that you can use in your terminal program.

Basic local workflow

If you follow these steps you should have a fully functional Git repository that tracks changes for you, and that you can use to collaborate on code (or documents, or files, or anything) with other people.

  1. Download Git
  2. Using your terminal, go to the directory that you want to track with Git

  3. Type the following commands:

    1
2
3

    git init
git add .
git commit -am 'Example commit message'

Repeat lines 2 + 3 every time you make a significant change in your repository

Explanation

The above workflow lets you harness a lot of what makes Git powerful without being too complicated.

  • git init initialises a repository in the directory you’re currently in. If your computer lets you see hidden directories, you’ll notice that a directory called .git/ has been createdâ€”this is where Git stores your changes (and some other information to keep your repository running). Once you initialise a repository, Git will attempt to keep track of all subdirectories and files inside the repo.
  • git add tells your repository to take notice of a file, which you specify as the next argument to add (e.g. git add example.txt). By using git add ., you are telling Git to track everything in the current directory you’re in.
  • Once you’ve told Git to pay attention to the files in your repo, you can start committing changes. A commit is like a snapshot of your repo, and you can make as many as you want.

Sync with a Github repository

Even more that it’s version control functionality, Git is used to collaborate on code. A nice place to set up a remote Git repository is on Github.

  1. Sign up for a Github account.
  2. Add your public SSH key to Github. This link includes guides on how to generate an SSH key if you don’t have one already.
  3. Create a new repository.
  4. Find your remote repository URL. It should look something like: git@github.com:username/repo_name.git
  5. git remote add origin YOUR_REMOTE_URL
  6. If you haven’t yet, make your first commit (lines 2 + 3 in Basic local workflow).
  7. git push origin master

Explanation

After you’ve done this, your repository knows that it has a remote that it can send changes to (and pull changes from, as we’ll see later). It also knows that the default (master) branch on your computer corresponds to the master branch on the remote repository.

  • git remote includes a bunch of commands to manage remote Git repositories. Most often you’ll just use it to add your main remote (often called origin), and that’s that.
  • git push origin master tells Git that the branch you’re currently on (which is master by default), should also be called master on the server.
  • When you use git push, it remembers the remote and branch that you previously specified and synchronises your local repo with your remote one.

Keep your work in sync with the remote
