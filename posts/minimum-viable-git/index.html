Minimum Viable Git

This is a collection of commands and short explanations of what they do. It should have you using the most common 80% of Git chop chop.

Words that are marked like this are commands that you can use in your terminal program.

Basic local workflow

If you follow these steps you should have a fully functional Git repository that tracks changes for you, and that you can use to collaborate on code (or documents, or files, or anything) with other people.

  1. Download Git
  2. Using your terminal, go to the directory that you want to track with Git
  3. git init
  4. git add .
  5. git commit -am 'Your commit message'
  6. Repeat steps 4 + 5 every time you make a significant change in your repository

Explanation

The above workflow lets you harness a lot of what makes Git powerful without being too complicated.

  • git init initialises a repository in the directory you’re currently in. If your computer lets you see hidden directories, you’ll notice that a directory called .git/ has been created—this is where Git stores your changes (and some other information to keep your repository running). Once you initialise a repository, Git will attempt to keep track of all subdirectories and files inside the repo.
  • git add tells your repository to take notice of a file, which you specify as the next argument to add (e.g. git add example.txt). By using git add ., you are telling Git to track everything in the current directory you’re in.
  • Once you’ve told Git to pay attention to the files in your repo, you can start committing changes. A commit is like a snapshot of your repo, and you can make as many as you want.

Sharing your work

So you’ve managed to set up a repo and are diligently keeping snapshots of your work. The next thing you’d like to do is share it with someone else. For this you’ll need a remote server that you have permission to access.

Setup

Only do this once per repo:

  1. git remote add origin YOUR_REMOTE_URL
  2. git push origin master

Workflow

Do this each time you’d like to make a change and push it to your remote:

  1. git add .
  2. git commit -am 'Example commit message'
  3. git push

Explanation

  • git remote includes a bunch of commands to manage remote Git repositories. Most often you’ll just use it to add your main remote (often called origin), and that’s that.
  • git push origin master tells Git that the branch you’re currently on (which is master by default), should also be called master on the server.
  • When you use git push, it remembers the remote and branch that you previously specified and synchronises your local repo with your remote one.
